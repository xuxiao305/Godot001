# 0008 — 自爆跳（Self-Splash Jump）：火箭跳是自爆涌现，不是后坐力

## Context

武器原型早期讨论中默认认为"火箭跳 = 后坐力反向推飞玩家"。Quake 玩家社区"rocket jump"在工程实现上有两种来源：

- 路径 A：开火瞬间的**后坐力**反推玩家
- 路径 B：玩家朝脚下开枪，火箭/榴弹在脚下爆炸，**爆炸的径向冲量**推飞玩家自己（"self-splash"）

本项目早期讨论混用，导致两个隐患：
1. 调参方向错 —— 后坐力调大会让"开枪 = 在地上漂移"，质感拉胯
2. 设计语言模糊 —— 之后讨论"火箭跳要不要做" 时两人在说不同的事

后坐力实际量级测算（武器 spec §4.1）：
- `recoil_impulse = 2 N·s`，角色 mass = 1.3 kg
- 反向速度 ≈ 1.5 m/s
- 静止地面起跳实际跳高 ≈ 0.045 m（v² / 2g，g=25）

爆炸推飞量级（武器 spec §4.3 + ADR-0007 引用值）：
- `peak_impulse = 12 N·s` 中心、`R = 3 m` 线性衰减
- 角色 mass = 1.3 kg、爆点距脚 ≈ 0.5 m → 衰减后冲量 ≈ 10 N·s → 反向速度 ≈ 7.7 m/s
- 起跳实际可达高度 ≈ 1.2 m

两个数量级差异（27 倍速度差，27 倍跳高差）。

## Decision

**承认两条物理路径独立存在，明确命名分离，且本项目的"火箭跳"专指自爆跳（路径 B）。**

| 术语 | 物理来源 | 量级 | 玩法意义 |
|---|---|---|---|
| **后坐力 (Recoil)** | 开火瞬间反向冲量施加在玩家身上 | 2 N·s → 1.5 m/s | 质感反馈、轻微空中调位 |
| **自爆跳 (Self-Splash Jump)** | Projectile 在脚下爆炸，Effect 的 ForceField 推飞玩家自己 | 10-12 N·s → 7-9 m/s | 替代/增强跳跃，可达 1-2 m 高度 |

实现层面：

- 后坐力是 Weapon 类的一个属性 `recoil_impulse`，可独立开关（用于 A/B 测试质感）
- 自爆跳来自 ForceField 的 `affect_player = true` —— 这是 [ADR-0007](0007-effect-dual-channel.md) Effect 双通道架构的**核心 invariant**，不是额外特性

## Rationale

**为什么必须命名分离：**

- 调参方向相反：后坐力应该小（避免开枪 = 漂移），自爆跳由爆炸本身参数控制
- 设计取舍正交：可以保留后坐力但禁止自爆跳（`affect_player = false`），反之亦然
- 玩家可读性：未来加 tutorial / 介绍机制时，两者在 UI / 文字上需要分别讲

**为什么自爆跳走 `affect_player = true` 而不是单独路径：**

- ForceField 对所有 dynamic body 一视同仁是 [ADR-0001 内在发动机派](0001-inner-engine-school.md) 的核心承诺 —— 玩家就是 dynamic body
- 加 `affect_player` 开关是为了**调试** A/B 测试用，不是为了"特例排除玩家"
- 任何"自爆跳"额外路径 = 重写一遍 ForceField 逻辑 = 违反"自然涌现"原则

**为什么自爆跳是 feature 不是 bug：**

- 与 [ADR-0006 二段跳 = 喷气背包](0006-double-jump-as-jetpack.md) 同源 —— 角色机动性扩展通过物理叙事获得，而非魔法
- "朝脚下开枪获得高度"在物理直觉上完全合理，玩家会自然发现
- 与 [项目愿景 setpiece](../项目总览.md) 兼容：未来"被爆炸推飞过缺口"是设计目标，自爆跳是同一机制的玩家主动版

**为什么不做"屏蔽自爆"的玩法保护：**

- 屏蔽会让"爆炸推飞玩家"的所有 setpiece 失去物理一致性 —— 与项目核心承诺矛盾
- "自爆跳变成主玩法、喧宾夺主"在 demo 阶段是好事（验证物理统一架构）
- 完整游戏阶段如需平衡，调参（爆炸量级、cooldown）即可，无需架构改动

## Consequences

### 对武器 spec

- §4.1 武器属性表明确：`recoil_impulse` 与 `affect_player` 是两个独立参数，分别控制两条路径
- §4.6 章节标题应从"后坐力对 3C 的影响"改为"对玩家的双路径影响（后坐力 + 自爆跳）"
- §6 验证标准应分别测两条路径

### 对 ADR-0007

- ForceField 的 `affect_player = true` 在 ADR-0007 中已隐含；本 ADR 提供其玩法理由

### 对未来武器扩展

- 任何"对玩家产生位移效果"的新机制（钩爪枪、引力枪反冲）都应明确归类到这两路径之一，或承认是第三路径（如绳张力）

### 命名洁癖

- 项目文档全文 ban "火箭跳"作为指代（多义），统一用"自爆跳"
- "后坐力跳"也不用 —— 后坐力量级根本不够"跳"

## Open Questions

- 量级是否要再分级（手枪后坐 1 N·s、火箭炮后坐 2 N·s）—— v1 不做差异化
- 自爆跳是否需要 cooldown 兜底（玩家无限自爆飞）—— v1 不做；后续被武器自身 cooldown 限制
- 角色受爆炸推飞后的"无敌帧"（避免被同一爆炸链式反复打）—— v1 不需要（v1 无伤害到玩家概念，只有冲量）
