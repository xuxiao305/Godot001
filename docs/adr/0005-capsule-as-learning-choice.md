# 0005 — Capsule 作为学习先行的角色形状

## Context

讨论角色形状时识别出 Capsule 在 Box2D 下的弹跳风险（弧面碰台阶角 → 法线偏转 → 接地态闪烁 + 水平减速冲量 + fixed_rotation 反作用力放大）。"Box 主体 + 圆形脚趾" 在物理上更优，但更复杂。

## Decision

v1 采用 **Capsule** 形状（同文档原方案 §4.1），不预先升级到 Box+脚趾。

接地检测采用 **capsule fixture 的 Box2D contact + 接触法线 y 分量检查**（不用 raycast），1 帧防抖接口实现但**默认关闭**。

## Rationale

- 本项目是学习项目，需要**真机感受 Capsule 的弹跳现象**而不是预防性规避
- raycast 会过滤掉接地态闪烁信号 → 学习不彻底；contact + 法线让所有症状浮现到 debug 面板
- 防抖默认关闭 → 直观对比"加防抖前后"差异，理解每条魔法的代价
- 跟 [项目总览 §2.5](../../项目总览.md) "纯物理优先，魔法后加" 完全一致

## Consequences

- 接触法线阈值（接地判定）需要在 Capsule 下花时间调，比 Box+脚趾松（约 45-60°），可能引入"贴墙误判接地"的边缘 case，调试这些是预期学习产物
- v1 完成后若决定切换到 Box+脚趾，接地检测代码（contact + 法线）**无需改动**，只换 fixture 几何
- 测试关卡需要包含**台阶序列**（不在文档原 §4.10 列表里），用于直接观察弹跳
