# 0012 — 项目方向从编排派转向涌现派

## Context

项目原定方向（[[项目总览]] §2.1）：

> **编排 > 系统**
> 参考 Half-Life 2 / Uncharted 的脚本物理 setpiece，**不是** Noita / Worms 的纯涌现派。
> **含义**：物理首先服务于"奇观时刻"，其次才是机制深度。

由此推导出的红线（§3、§8）：
- ❌ 不做真流体模拟，用高度场近似替代
- ❌ 不做像素材料系统
- 收敛 demo 愿景全部为"setpiece"导向（失控矿车 / 塌陷废墟 / 沉没的船）

**触发本次转向的契机：** 在 [[想法仓库]] 收集三条新想法时，I-001"统一物理 + 粒子水体"明确违反上述红线。讨论后用户意识到自己真正想做的是涌现派路线，而不是编排派。

## Decision

**项目方向从编排派转向涌现派（emergent simulation）。**

具体含义：
1. **设计哲学反转**：系统 > 编排。物理首先服务于"机制深度与统一性"，奇观时刻作为机制涌现的副产物，而不是反过来用机制服务奇观。
2. **取消"真流体"红线**：粒子水体、密度浮沉、流体动量传递等纳入可探索范围；高度场水降级为"性价比备选"，不再是默认方案。
3. **取消"像素材料"红线**：在算力允许的范围内可探索（如沙、雪、岩浆的粒子化），但不强求 Noita 级密度。
4. **参考坐标系**：从 Half-Life 2 / Uncharted **改为** Noita / Worms / Teardown / Besiege。
5. **收敛愿景调整**：从"setpiece demo"改为"涌现系统沙盒"。具体候选见 §Rationale 末。

## Rationale

**为什么转：**
- 涌现派与"物理统一优先"（[[项目总览]] §2.5）和"内在发动机派"（[ADR-0001](0001-inner-engine-school.md)）天然同源——两者都强调"无特例、统一规则、行为自发涌现"。编排派路线下，§2.5 反而是被压抑的：奇观时刻往往需要脚本注入（"在第 30 秒触发塌方"），破坏统一性。
- 学习项目的最大收益是"探索硬问题"。粒子流体 / 软体 / 材料系统是 Box2D 之外的硬骨头，避开它们等于回避项目最有学习价值的部分。
- 涌现派天然允许"做小做散"——每个机制独立可玩，不必拼出一个完整 setpiece 才有意义。这与 §2.3 范围克制和原型驱动方法学高度一致。

**保留不变：**
- [ADR-0001 内在发动机派](0001-inner-engine-school.md) —— 角色控制器架构不变，反而被强化（角色 = 物理体，与世界统一规则）
- §2.5 物理统一优先 —— 不变，反转的是"编排 vs 系统"的优先顺序，不是"物理 vs 魔法"
- §2.6 反物理玩法优先级方法学 —— 不变
- 武器系统 ADR-0007~0010 的"单向依赖 / Effect 双通道 / 三元组分解" —— 这些本来就是涌现派友好的架构

**新方向的候选收敛愿景（替代原 §7）：**
- 🥇 **物理沙盒**：玩家在一个混合了破坏、流体、可拼装载具、武器的房间里自由实验。无关卡设计，单纯的玩具盒。Noita / Besiege 的 Sandbox 模式风格。
- 🥈 **小型涌现挑战**：一组小关，每关给一个目标（如"让那个箱子沉到水底"、"造一辆能爬上斜坡的车"），玩家用统一物理工具集解决。Zachtronics 风格但物理化。
- 🥉 **机制独立 demo 集**：每个机制（粒子水、组装载具、破坏、武器）有自己的小展厅，进去就能玩。Garry's Mod 早期风格。

## Risk

- **范围失控风险**：涌现派天然容易"什么都想做"。对策：仍然 YAGNI、原型驱动，每个机制做完再决定下一个。
- **demo 化困难**：涌现派 demo 难"录视频展示"，因为没有脚本时刻。对策：可以为每个机制录"玩家自由实验"片段，而不是"剧情演示"。
- **本次转向本身可能再翻**：项目还在阶段 1，方向可逆。如果未来发现涌现派对 1 人项目过于膨胀，可再开 ADR 修订。本 ADR 不是终局承诺。
