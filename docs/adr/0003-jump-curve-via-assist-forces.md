# 0003 — 用角色辅助力实现跳跃曲线特征，而非状态化重力

## Context

3C 设计原方案用状态切换重力（`gravity_fall_multiplier=1.8`，`apex_gravity_multiplier=0.4`）实现"飘感 + 砸感"。这是 Celeste / Hollow Knight 派的标准配方。但状态切换重力违反 [ADR-0001](0001-inner-engine-school.md) 的"统一物理"哲学 —— 世界重力随角色状态变化没有物理意义，破坏所有物理体规则一致性。

## Decision

世界重力**恒定**。所有跳跃曲线特征通过**作用在角色身上的辅助力**实现：

| 期望效果 | 实现方式 |
|---|---|
| 顶点飘 | 顶点附近向上的小辅助力（持续 ~0.08 s） |
| 快落 | 下落开始时向下的小辅助力 |
| 上升慢 | 不加额外力，纯抛物线 |

**关键规则**：角色被外力推到空中时（爆炸、被怪打），**关闭这些辅助力** → 角色变回纯物理体，下落严格遵守重力。

## Rationale

- 世界重力恒定 → 玩家、敌人、碎片、流体等所有物理体规则一致
- "角色辅助力"语义清晰：发动机在不同状态下输出不同的辅助力，符合"内在发动机"本体论
- "主动跳跃 = 引擎介入" vs "被动飞行 = 纯物理"的世界观完美对应，未来 setpiece 受益

## 范围

**v1 不实现这些辅助力**，先按真抛物线跑。遵守 [项目总览 §2.5](../../项目总览.md) "魔法后加" 原则。真机测试若手感不足再启用，启用接口预留。
