# 0004 — 空中控制：弱喷气 + 无阻力 + 跳跃距离由助跑决定

## Context

Force-based 派下，原文档 §4.3 的 `air_control_ratio = 0.65`（速度覆盖派术语）失去意义，需要重定义空中控制三要素：空中发动机推力大小、空中是否有阻力、空中 v_max 是否独立。

## Decision

| 项 | 取值 | 含义 |
|---|---|---|
| 空中发动机推力 | `F_max_air ≈ 0.5 × F_max_ground` | 玩家在空中能纠正方向，但比地面慢一倍 |
| 空中水平阻力 | **无**（无 linear damping，无 air drag） | 角色起跳后水平速度永久保持，除非主动反向喷气 |
| 空中 v_max | 同地面 v_max | v_max 是发动机额定转速属性，跟介质无关 |

## Rationale

- 空中无地面摩擦，若推力同地面，则空中转向反而比地面快 —— 违反直觉
- 无空气阻力 = 真空物理，最纯；加阻力就是"魔法"（[项目总览 §2.5](../../项目总览.md)）
- v_max 跟介质解耦 → 状态机最少，代码哲学一致

## Gameplay Grammar 影响（surprising 部分）

**横向跳跃距离由地面助跑决定，不由空中操控决定。**

| 玩家行为 | 横向位移 |
|---|---|
| 原地起跳 + 空中按方向 | 短（喷气推力小 + 时间短，最多到 v_max 的 30-50%） |
| 助跑到顶速 + 起跳 + 空中什么都不按 | 远（v_max × 滞空时间，全程保持） |
| 助跑到顶速 + 起跳 + 空中按反方向 | 中（半力反向喷气，慢慢减速甚至反向） |

这跟 Celeste / Hollow Knight 的"原地起跳能控全程"完全相反，跟 Trine / HL2 / Uncharted 的"跑跳一体" 一致 —— 跟项目愿景（神秘海域 setpiece）匹配。

## Consequences

- **关卡设计约束**：需要跨越的宽缺口必须前置足够长的助跑跑道，不能设计"必须原地大跳"的关卡
- **角色发动机的"额定" v_max** 在水平方向是软上限（自跑收敛），在垂直方向不存在（跳跃 vy 由冲量决定，跟 v_max 无关）
- **空中惯性可叠加**：跳起来后被爆炸再推一下，角色获得叠加冲量；落地后受摩擦衰减回 v_max 范围。这是 setpiece 戏剧感的连锁机制
