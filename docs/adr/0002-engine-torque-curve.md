# 0002 — 发动机转速曲线作为最大速度机制

## Context

"内在发动机派"（[ADR-0001](0001-inner-engine-school.md)）下，"最大速度"如果用 `clamp(v.x, -v_max, v_max)` 实现会变相退化成速度覆盖派，破坏 ADR-0001 的哲学。需要一个**物理意义自洽**的方案。

## Decision

最大速度由**发动机转速特性曲线**涌现，不直接 clamp 速度：

```
F_engine = F_max · softsign(v_target − v_current) · saturation(|v_target − v_current|)
```

- `v_target = ±v_max` 当玩家按方向键，`0` 当无输入
- `saturation` = 平滑衰减（sigmoid-like），不是矩形脉冲，模拟真实发动机扭矩曲线
- 玩家按反方向时发动机全力反向出力（即使当前已超速）
- 角色被外力推飞超过 v_max 时，发动机不强行回拉（出力 = 0），靠摩擦自然衰减

## Rationale

- 物理本体论清晰 —— "发动机有最大转速"是真实物理实体（汽车引擎、电动机皆如此），代码读者能直接接受
- 与"角色被推飞 → 允许瞬时超速 → 戏剧感"的项目愿景兼容
- 不破坏 ADR-0001 —— 全程都是力的求和，无速度覆盖

## Consequences

- 玩家可观察到"接近顶速时加速感放缓"，这是 feature 不是 bug
- 被爆炸 / 动平台 / 碰撞推飞时，角色速度可瞬时远超 v_max，自然衰减回正常区间。setpiece 演出依赖此性质
- 调参时 v_max 不再是"硬上限"，而是"自跑稳态速度" —— 受地面摩擦影响
- 调试面板需要同时显示 `v_current` 和 `v_target`，便于理解发动机出力状态
