# SPH 流体模拟（Smoothed Particle Hydrodynamics）

> **前置需求**：基本编程概念即可。本章自成体系，不依赖前几章。
> **阅读目标**：理解 SPH 的核心思想——如何用粒子模拟流体、Rapier+Salva 的实现方式、以及各种物理效果的直觉含义。

---

## 1. 问题：如何用粒子模拟水流

流体（水、熔岩、黏液）在游戏中很常见，但模拟起来非常困难。传统方法需要求解复杂的偏微分方程（Navier-Stokes 方程），计算量巨大。

SPH（Smoothed Particle Hydrodynamics，平滑粒子流体动力学）提供了一种巧妙替代方案：**把流体离散成几十上百个小粒子，每个粒子只与周围的邻居粒子互动**。用简单的"推拉"规则就能产生逼真的流体行为。

### 1.1 核心直觉：人群模型

想象一群人挤在房间里：

- **密度太高（太多人挤在一起）**：人群向外推 → **压力（Pressure）**
- **有人快速穿过人群**：会被周围的人拖慢 → **粘性（Viscosity）**
- **人群边缘的人**：被中心的人拉住，形成紧密的一团 → **表面张力（Surface Tension）**

SPH 流体粒子行为与此类似——每颗粒子根据周围邻居的状态计算受的力，然后像刚体那样更新速度和位置。

### 1.2 Rapier + Salva

Rapier 本身不处理流体。流体模拟由 **Salva** 库提供（Salva 是 Dimforge 组织的配套库，专门做 SPH 流体模拟）。godot-rapier-physics 将 Salva 集成到 Godot 中。

流体相关文件：
- `src/rapier_wrapper/fluid.rs`：Salva 的封装层，管理 `LiquidWorld` 的创建、更新、查询
- `src/fluids/fluid_effect_2d_*.rs`：各种物理效果的 Godot 资源包装（表面张力、粘性、弹性）

## 2. SPH 的工作原理

### 2.1 从连续到离散

在现实世界中，水是连续的——你可以取任意小的水滴。但计算机处理不了"无限"的信息，所以 SPH 的核心思想是：**用有限个粒子来近似连续的流体**。

每个粒子代表流体的一小部分。粒子的半径决定了"平滑尺度"——粒子越多、半径越小，模拟越精确，但也越慢。

### 2.2 密度计算：加权邻居求和

每个粒子的局部密度不是直接给出的，而是通过**观察周围邻居**计算出的：

```
我的密度 = 所有邻居粒子的（质量 x 该邻居处的核函数值）的总和
```

**直觉**：如果你周围有很多邻居粒子 → 你所在区域的密度高。如果你周围空空荡荡 → 密度低。

平滑核函数（Smoothing Kernel）的作用是让**离得近的邻居贡献更大**，离得远的邻居影响迅速衰减到零。就像一个"影响力范围"——超出半径的邻居完全不计入。

### 2.3 压力：密度差产生推力

压力是 SPH 中最重要的力。核心规则很简单：

```
如果我的密度 > 静止密度 → 产生向外推力（周围太挤了！）
如果我的密度 < 静止密度 → 产生向内拉力（周围太空了！）
```

**静止密度（Rest Density, `density0`）**是流体的自然密度。水的密度约 1000 kg/m^3——对于 SPH 粒子，这个值定义了"正常拥挤程度"。大于它就推，小于它就拉。

**日常类比**：地铁车厢里，人越多，你被推得越厉害。当车厢半空时，大家自然散开，直到形成均匀的间距。

### 2.4 粘性（Viscosity）：速度差产生拖拽

粘性描述流体的"黏稠度"：

- **水**：粘性低，粒子可以自由滑动
- **蜂蜜**：粘性高，粒子互相拉扯，整体流动缓慢
- **空气**：粘性极低，几乎无阻力

SPH 中的粘性力：如果我和邻居的速度不同 → 产生一个力让我"跟上"邻居的速度。邻居比我快 → 拉动我。邻居比我慢 → 拖慢我。

### 2.5 表面张力（Surface Tension）：让水滴成为球

表面张力是分子级别的内聚力在宏观上的表现。在 SPH 中：

- **内部粒子**：四面八方都是邻居，受力均匀，净力为零
- **表面粒子**：外部没有邻居 → 产生净向内拉力 → 流体倾向于最小化表面积 → 形成球形水滴

**日常类比**：水龙头下形成的水滴总是球形的。荷叶上的露珠聚成小球。这就是表面张力的效果——表面分子被内部拉向中心。

## 3. Salva 中的物理效果实现

Salva 将物理效果建模为"非压力力"（Nonpressure Forces）——独立于基本压力计算的额外力。每种效果是一个可插拔的组件，通过 `fluid.nonpressure_forces.push()` 添加到流体上。

在 godot-rapier-physics 中，这些效果被包装为 Godot Resource：

| Godot 类 | Salva 类型 | 用途 |
|---|---|---|
| `FluidEffect2DViscosityArtificial` | `ArtificialViscosity` | 人工粘性 |
| `FluidEffect2DViscosityDFSPH` | `DFSPHViscosity` | 高稳定粘性 |
| `FluidEffect2DViscosityXSPH` | `XSPHViscosity` | 平滑粘性 |
| `FluidEffect2DSurfaceTensionAkinci` | `Akinci2013SurfaceTension` | 表面张力（Akinci 模型） |
| `FluidEffect2DSurfaceTensionHe` | `He2014SurfaceTension` | 表面张力（He 模型） |
| `FluidEffect2DSurfaceTensionWCSPH` | `WCSPHSurfaceTension` | 表面张力（WCSPH 模型） |
| `FluidEffect2DElasticity` | `Becker2009Elasticity` | 弹性（固液混合） |

## 4. 三种粘性方法对比

### 4.1 Artificial Viscosity（人工粘性）

**理念**：直接在粒子之间按速度差施加拖拽力。

- 优点：计算最快，实现最简单
- 缺点：可能过度消散能量（流体比实际更"黏"）
- 参数：`fluid_viscosity_coefficient`（流体内部粘性系数）、`boundary_viscosity_coefficient`（与边界碰撞的粘性）

适合：效果不关键的场景、移动端低性能目标

### 4.2 DFSPH Viscosity（Divergence-Free SPH）

**理念**：基于"不可压缩性"的严格约束来计算粘性力。

- 优点：极其稳定，没有数值震荡，支持高粘性值
- 缺点：计算量大于人工粘性
- 参数：`fluid_viscosity_coefficient`

适合：需要精确物理行为的场景（如工程可视化、高质量流体效果）

### 4.3 XSPH Viscosity（eXtended SPH）

**理念**：通过对邻居速度进行加权平均来平滑速度场。

- 优点：产生非常平滑的流动效果（粒子运动优雅流畅）  
- 缺点：本身不提供真实的物理粘性
- 参数：`fluid_viscosity_coefficient`、`boundary_viscosity_coefficient`

适合：视觉效果优先的场景（粒子不会聚成难看的不规则团块）

## 5. 三种表面张力方法对比

### 5.1 Akinci (Akinci2013SurfaceTension)

**理念**：基于"色函数"（color function）——计算每个粒子的"表面程度"，对表面粒子施加向内拉力。

- 优点：计算高效，适合实时
- 参数：`fluid_tension_coefficient`（表面张力系数）、`boundary_adhesion_coefficient`（与边界壁的附着）

### 5.2 He (He2014SurfaceTension)

**理念**：改进版的表面张力，通过更精确的表面粒子检测减少非物理振荡。

- 优点：比 Akinci 更稳定，产生的表面更光滑
- 参数：同 Akinci

### 5.3 WCSPH Surface Tension

**理念**：将表面张力直接融入压力计算中（Weakly Compressible SPH 框架）。

- 优点：不需要单独的"表面检测"步骤
- 参数：同上

## 6. 弹性（Elasticity）——仅 Becker2009 模型

弹性效果由 `Becker2009Elasticity` 提供，这是当前 godot-rapier-physics 集成中**唯一可用的弹性模型**。它将流体粒子转化为半固态——介于流体和固体之间。通过"杨氏模量"（Young's Modulus）和"泊松比"（Poisson Ratio）控制：

- **Young's Modulus（杨氏模量）**：材料有多硬。高 = 像石头，低 = 如果冻
- **Poisson Ratio（泊松比）**：挤压一个方向时，其他方向的膨胀程度。橡胶约 0.5，软木约 0

用法：给流体添加弹性效果 → 粒子和粒子之间像有小弹簧连接 → 可以模拟果冻、软体、橡胶等非牛顿流体。

## 7. 流体与刚体的交互

SPH 粒子不能孤立存在——它们需要与场景中的刚体（地面、墙壁、浮动物体等）进行双向交互。

### 7.1 边界碰撞（Boundary Collision）

流体粒子碰到刚体碰撞体时，被当作与"固体边界"的碰撞。粒子被推出碰撞体表面，保证流体不会渗透进刚体内部。边界粘性参数（`boundary_viscosity_coefficient`）控制流体沿壁面滑动的阻力——值越大，流体越"粘"在壁面上。

### 7.2 浮力与拖拽（Buoyancy and Drag）

当刚体浸入流体时，受到两个力：
- **浮力**：与刚体排开的流体体积成正比（阿基米德原理）。排开越多流体 → 浮力越大 → 物体上浮。密度小于流体密度的物体会上浮（木板浮在水上），密度大于流体的会下沉（石头沉入水底）。
- **拖拽力**：流体对运动刚体的阻力，方向与刚体运动方向相反，大小与速度平方成正比。

在 Salva 中，这些交互力的计算通过流体密度场查询和边界粒子采样实现。对于性能敏感的场景，可以用简化模型（如给浸没刚体施加恒定的上浮力和线性阻尼）替代全 SPH 查询。

### 7.3 双向耦合

流体推刚体（浮力、拖拽），同时刚体也推流体（刚体运动改变了流体粒子的边界条件）。这种双向耦合使得系统即使在简化模型下也能产生真实的行为——比如活塞推动流体、漂浮物随波浪起伏。

## 8. 性能考量

SPH 对粒子数量非常敏感：

- 粒子数量翻倍 → 邻居搜索翻倍 → 密度计算至少翻倍 → 整体性能可能下降 4 倍以上
- 50-200 颗粒子：移动端可稳定 60 FPS
- 500-1000 颗粒子：PC 端 60 FPS
- 2000+ 颗粒子：需要降低粒子半径或接受更低的帧率

Salva 内部使用空间哈希网格（Spatial Hashing Grid）加速邻居搜索——粒子只需要检查相邻网格内的其他粒子，而不是全局遍历。

优化建议：
- 使用较大的粒子半径（减少粒子数量）
- 只在需要时启动物理效果（粘性/表面张力不是免费的）
- 限制流体在屏幕内（远处/不可见的流体 = 浪费）
