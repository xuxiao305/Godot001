# 刚体动力学（Rigid Body Dynamics）

> **前置需求**：建议先阅读 [00-rapier-architecture.md](./00-rapier-architecture.md) 了解整体架构。
> **阅读目标**：理解质量、速度、力、冲量、阻尼、睡眠等核心概念，以及 Rapier 如何模拟物体运动。

---

## 1. 问题：物体为什么会动

想象桌上有一本书。它不会自己动——需要你**推**它（施加力），它才会移动。如果书很重（质量大），你需要更用力推才能让它动起来。如果书很轻，轻轻一推就飞出去了。

艾萨克-牛顿爵士在三百多年前就把这些直觉总结成了三条定律。物理引擎正是把这三条定律变成计算机能理解的数学公式。

---

## 2. 质量（Mass）和惯性（Inertia）

### 2.1 质量不是重量

日常生活中我们说"这本书 500 克"，那是**质量**。重量是质量乘以重力加速度——在月球上，书的质量没变，但重量只有地球的 1/6。

在物理引擎中：
- **质量（Mass）**：衡量一个物体"抗拒运动变化"的能力。质量越大，同样的力产生的加速度越小。
- **惯性（Inertia）**：质量的旋转版本。惯性越大，同样的扭矩产生的角加速度越小。

**日常类比**：推一个空购物车和推一个装满东西的购物车。你认为哪个更难推动？质量大的那个更难推动——这就是"质量 = 对运动变化的抗拒"。

### 2.2 Rapier 中质量的计算

在 Rapier 中，刚体的质量不是手动设置的，而是由附着在上面的**碰撞体形状和密度**自动计算的（参考 `src/dynamics/rigid_body_components.rs` 中的 `RigidBodyMassProps`）。

在 2D 中，"体积"实际上是面积，质量 = 面积 x 密度。例如：
- 一个边长为 1m、密度为 1.0 的正方形，质量 = 1 x 1 x 1.0 = 1 kg
- 一个半径为 0.5m、密度为 2.0 的圆形，质量 = pi x 0.5^2 x 2.0 ~= 1.57 kg

如果需要额外质量（比如内部装的货物），可以用 `set_additional_mass()` 添加。

在 2D 中，旋转惯性是一个**标量**（只有一个旋转轴 Z）。用 `RigidBody::lock_rotations(true)` 将惯性设为无穷大，阻止旋转。

## 3. 速度（Velocity）

### 3.1 两种速度

- **线速度（Linear Velocity, `linvel`）**：物体质心在空间中移动的速度。是一个向量，既有大小（速率）也有方向。
- **角速度（Angular Velocity, `angvel`）**：物体绕自身中心旋转的速度。在 2D 中是一个标量（弧度/秒），正 = 逆时针，负 = 顺时针。

### 3.2 如何理解角速度

找一个旋转的物体——比如正在转的硬币。角速度 1.0 rad/s 意味着每秒转大约 57.3 度。

在 Rapier 中，线速度和角速度存储在 `RigidBodyVelocity` 结构体中（`linvel: Vector, angvel: AngVector`）。

### 3.3 如何改变速度

有**三种方式**改变速度，各有用处：

| 方式 | 特点 | 使用场景 |
|---|---|---|
| `set_linvel()` / `set_angvel()` | 直接设置，覆盖原速度 | 角色移动、平台控制 |
| `add_force()` / `add_torque()` | 持续累加，每帧清零 | 推进器、风、磁力 |
| `apply_impulse()` / `apply_torque_impulse()` | 瞬间改变，即刻生效 | 跳跃、爆炸、被击中 |

直接设置速度**不经过物理模拟**，适合需要精确控制的物体（如玩家角色）。施加力是符合物理定律的方式——同样的力对重物影响小，对轻物影响大。

## 4. 速度积分（Velocity Integration）：从速度到位置

### 4.1 最简单的想法

如果你的 GPS 每秒钟报一次你的速度，你怎么知道 5 秒后你在哪里？

如果你在每秒开始时更新位置：`新位置 = 旧位置 + 当前速度 * 1秒`。

这就是 Rapier 使用的方法——**半隐式欧拉积分**（Semi-implicit Euler）。与其被名字吓到，不如记住：

```
新速度 = 旧速度 + (力 / 质量) * dt
新位置 = 旧位置 + 新速度 * dt
```

注意：**先更新速度，再用新速度更新位置**。这和"先用旧速度更新位置，再更新速度"的显式欧拉相比，更稳定。

### 4.2 dt 是什么

dt（delta time）是两次物理更新之间的时间间隔，通常为 1/60 秒 ≈ 0.0167 秒。

**关键直觉**：速度单位是"米/秒"，乘以 dt（秒）得到"米"——这段距离加上原位置，得到新位置。

Rapier 在 `RigidBodyVelocity::integrate()` 方法中实现这个计算（`src/dynamics/rigid_body_components.rs`）。

### 4.3 力 vs 冲量

- **力（Force）**：持续作用的推/拉，单位是牛顿（N）。比如重力每帧都对物体施加 `mass * 9.81` 的力。Rapier 中通过 `add_force()` 施加的力会**在每帧结束时清零**，所以必须在每帧重新施加。

- **冲量（Impulse）**：瞬间的动量改变，单位是 千克-米/秒。比如碰撞就是冲量的自然例子。Rapier 中通过 `apply_impulse()` 施加的冲量**立即改变速度**：`linvel += impulse / mass`。

**日常类比**：
- 力 = 持续推箱子 = 慢慢加速
- 冲量 = 踢箱子一脚 = 瞬间弹出去

## 5. 阻尼（Damping）：物体为什么会停下来

### 5.1 物理直觉

在现实世界中，滑动的箱子最终会停下——因为有**摩擦力**和**空气阻力**。在物理引擎中，这些耗散力的精确模拟计算量巨大，所以我们用一个简化的模型——阻尼。

### 5.2 阻尼的数学

Rapier 的阻尼公式（参见 `RigidBodyVelocity::apply_damping()`）：

```
新速度 = 旧速度 / (1 + 阻尼系数 * dt)
```

这意味着：
- `linear_damping = 0.0`：没有衰减（太空中的运动）——永远不减速
- `linear_damping = 0.1`：轻微衰减（空气中的运动）——缓慢减速
- `linear_damping = 1.0`：明显衰减（水中的运动）——迅速减速

**重要**：阻尼是**指数衰减**。如果 `linear_damping = 0.1, dt = 1/60`，每秒速度乘以约 `1 / (1 + 0.00167) = 0.9983`。60 帧后速度为原来的 0.9983^60 ≈ 90%。所以 damping = 0.1 不会让物体瞬间停止，而是平滑减速。

### 5.3 Rapier 支持两种阻尼

- `linear_damping`：线速度衰减
- `angular_damping`：角速度衰减

通过 `RigidBodyBuilder::linear_damping(0.1)` 设置，或运行时调用 `set_linear_damping(0.1)`。

## 6. 睡眠机制（Sleeping）：省电模式

### 6.1 问题

在一个关卡中，可能有几百个物体——其中大部分堆在地上不动。每帧都为它们做碰撞检测和速度积分毫无意义。

物理引擎的解决方案：让不动的物体"睡觉"。睡觉的物体不参与模拟（不检测碰撞、不积分速度），直到被"叫醒"。

### 6.2 Rapier 的睡眠判断

当一个刚体的线速度和角速度同时低于阈值并持续一定时间，它就可以睡觉。关键参数（定义在 `RigidBodyActivation` 结构体中）：

| 参数 | 默认值 | 含义 |
|---|---|---|
| `normalized_linear_threshold` | 0.4 | 线速度低于此值视为"静止"（会乘以 `length_unit`） |
| `angular_threshold` | 0.5 rad/s | 角速度低于此值视为"静止" |
| `time_until_sleep` | 2.0 秒 | 必须在阈值以下保持这么久才能睡觉 |
| `sleeping` | false | 当前是否在睡觉 |

### 6.3 岛屿检测与唤醒传播

睡眠不是单个物体的事。如果两个物体通过关节连接，一个醒了，另一个也必须醒。Rapier 使用**岛屿（Island）**管理这种关联。

唤醒的"涟漪效应"：
1. 一个物体被碰撞或施加力 → 唤醒
2. 与该物体有接触的物体 → 也唤醒
3. 与这些物体通过关节连接的物体 → 也唤醒
4. 传播直到所有相关的物体都醒了

这个过程由 `IslandManager` 和 `RigidBodyActivation` 配合实现。

### 6.4 什么时候禁用睡眠

大多数物体会自然睡觉。只有以下情况才需要 `RigidBodyBuilder::can_sleep(false)`：
- 始终需要射线检测的物体
- 静止时也有行为逻辑的物体（如定时器触发）
- 需要极其精确接触力的物体

## 7. 刚体类型

Rapier 有三种刚体类型（`RigidBodyType`）：

| 类型 | 受重力/力影响 | 能推动其他物体 | 能被其他物体推动 | 典型用途 |
|---|---|---|---|---|
| `Dynamic` | 是 | 是 | 是 | 箱子、子弹、布娃娃 |
| `Fixed` | 否 | 否 | 否 | 地面、墙壁、地形 |
| `KinematicVelocityBased` | 否 | 是 | 否 | 移动平台、电梯 |
| `KinematicPositionBased` | 否 | 是 | 否 | 动画控制的物体 |

Dynamic 是"完全物理模拟"的物体。Fixed 是"永远不动"的物体，质量为无穷大。Kinematic 是"你可以控制，但不会被推"的物体——它会推动 Dynamic 物体但不会被推回去，就像传送带上的箱子。

KinematicVelocityBased 通过直接设置速度来控制（`set_linvel`），而 KinematicPositionBased 通过设目标位置来控制（`set_next_kinematic_translation`），Rapier 会自动计算所需速度。
