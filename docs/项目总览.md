## 1. 项目愿景

做 **"涌现派 2D 物理沙盒"** —— 用统一的物理机制让丰富行为自发涌现：

- 粒子化流体（水、油、岩浆），与刚体真实互动（浮沉、冲击、溅起）
- 可破坏 / 可拼装 / 可改造的世界——所有方块、零件、载具用同一套物理规则
- 武器、爆炸、流体、结构都遵循"无特例的统一物理"，奇观从机制中涌现
- 玩家用一套统一工具集自由实验、组合、破坏、创造

**典型画面**：玩家拼出一辆战车，开进水里看它漂着前进；炸开一面墙，碎片砸入水池溅起水花；水从破口涌出冲走小箱子，水流推着角色与碎片继续演化。**没有脚本，全部是机制相互作用的结果。**

**参考坐标系：** Noita / Worms / Teardown / Besiege（涌现派），**不是** Half-Life 2 / Uncharted（编排派）。本项目曾走编排派路线，已于 2026-05-25 转向涌现派，详见 [ADR-0012](docs/adr/0012-pivot-to-emergent-simulation.md)。

## 2. 设计哲学

### 2.1 系统 > 编排
参考 **Noita / Worms / Teardown / Besiege** 的涌现派路线，**不是** Half-Life 2 / Uncharted 的脚本 setpiece。
**含义**：物理首先服务于"机制深度与统一性"，奇观时刻作为机制涌现的副产物。
**历史**：项目曾走编排派路线，已于 2026-05-25 转向涌现派，详见 [ADR-0012](docs/adr/0012-pivot-to-emergent-simulation.md)。

### 2.2 原型驱动
每个物理子系统先做成独立 demo，跑通后再合体。避免"全做了但每个都半成品"的陷阱。

### 2.3 范围克制
- 单原型聚焦，YAGNI
- 不为未来可能的功能预留架构
- 跑通 → 学到 → 移到下一个

### 2.4 不追完整游戏体验
不需要剧情、UI、存档、关卡量；但每个 demo 要"足够流畅地撑起一段体验"。

### 2.5 物理统一优先，魔法后加
角色与世界的所有行为都先尝试用**统一物理**（力、冲量、摩擦、恒定重力）实现；只有当真机测试证明纯物理无法达到所需手感时，才考虑加"**魔法**"（状态化重力、速度覆盖、特例修正、输入容忍）。
**含义**：宁可"看起来朴素但内部一致"，不要"看起来花哨但靠特例堆砌"。
**实施层面**：每条魔法都要明确为"魔法"并能开关，方便回退到纯物理验证差异。

### 2.6 反物理玩法的优先级方法学
遇到看似违反纯物理的玩法机制（二段跳、Dash、墙跳、钩爪、抓边等），按以下优先级处理：

1. **优先：物理化包装** —— 给反物理动作一个**物理叙事**（如二段跳 = 喷气背包反作用力，详见 [ADR-0006](docs/adr/0006-double-jump-as-jetpack.md)）。代码上仍是冲量，但本体论跟"内在发动机派"自洽
2. **其次：知觉一致性补偿** —— 如果是输入容忍类（coyote、buffer），按 [CONTEXT.md](CONTEXT.md) 的"知觉补偿"概念处理，不算魔法
3. **兜底：承认魔法** —— 包装不下的才纯魔法实现（状态化重力等）。必须有开关、可对比关闭后行为

## 3. 引擎约束（决定所有可行性边界）

### ✅ Box2D 白送（几十行代码就跑）
- 刚体堆叠 / 塌方 / 断裂
- 关节系统（绳、铰链、滑轨、齿轮、皮带）
- 浮力区域（不是真流体，但效果好）
- 弹道、爆炸冲击波
- 车辆、机械、布娃娃
- 摩擦、弹性、复杂碰撞

### ⚠️ Box2D 做不了（需造轮子，但涌现派下值得做）
- 真流体模拟（SPH、像素级液体）
- 软体形变 / 布料 / 果冻
- 像素材料系统（Noita 那种）

> **2026-05-25 后**：这些不再是回避项。涌现派方向下，它们是项目最有学习价值的硬骨头，详见 [ADR-0012](docs/adr/0012-pivot-to-emergent-simulation.md)。

### 实现策略
- **流体（首选）**：粒子化流体——大量小 dynamic body + 自定义粒子-粒子作用力，与刚体走统一碰撞解算。性能瓶颈需实测。
- **流体（备选）**：高度场水——若粒子方案性能崩盘则回退。视觉够用但放弃统一物理。
- **软体**：关节链 + 视觉插值（伪软体）——v1 不优先。
- **像素材料**：在算力允许的范围内可试，但不强求 Noita 级密度。

## 5. 模块依赖关系

```
                    ┌────────────────┐
                    │  家族 A: 3C    │ ← 所有其他模块的前置
                    └────────┬───────┘
                             │
        ┌────────────────────┼────────────────────┐
        ↓                    ↓                    ↓
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│ 家族 B: 破坏 │    │ 家族 C: 流体 │    │ 家族 D: 武器 │
└──────┬───────┘    └──────┬───────┘    └──────┬───────┘
       │                   │                   │
       └─────────┬─────────┴─────────┬─────────┘
                 ↓                   ↓
        ┌────────────────┐  ┌────────────────┐
        │ 家族 E: 载具/  │  │ 家族 F: 编排   │
        │     关节体     │  │   setpiece     │
        └────────────────┘  └────────────────┘
                                    ↑
                              (家族 F 是终态,
                          需要 A + 至少 1-2 其他)
```

**关键观察：**
- **A（3C）是所有的根** —— 必须先做
- B / C / D 互相独立，可并行（按兴趣挑顺序）
- D（武器/爆炸）和 B（破坏）天然耦合，但可以分开做完再合
- F（编排）需要 A + 至少 1-2 个其他家族就位

## 6. 路线图建议

### 阶段 1：地基（当前）
- 🔄 **原型 1：3C** —— 见 [[2026-05-24-3C-prototype-design]]

### 阶段 2：核心物理玩法（任选其一先做）
- ⬜ **原型 2A：可破坏方块墙** —— 测家族 B 核心（焊接关节断裂、分块破坏）
- ⬜ **原型 2B：高度场水 + 浮力** —— 测家族 C 核心（水面渲染 + 浮力区 + 涉水）
- ⬜ **原型 2C：物理子弹 + 爆炸** —— 测家族 D 核心（弹道 + 径向冲击 + 简单破坏）

### 阶段 3：组合原型
- ⬜ **原型 3A：可破坏 + 爆炸** = 2A + 2C 合体
- ⬜ **原型 3B：水中战斗** = 2B + 2C 合体
- ⬜ **原型 3C：钩爪 + 摆荡** = 3C v2 + Box2D 绳关节

### 阶段 4：沙盒整合
基于已有原型，把多个机制合到一个房间里自由实验（不是脚本 setpiece，而是机制相互作用的玩具盒）。具体收敛愿景见 §7。

> **何时进入下一阶段：** 当前阶段所有原型都达到"自己看着满意"即可。不追求完美，追求"学到了"。

## 7. 收敛 demo 愿景（候选）

涌现派方向下，"demo"指机制相互作用的玩具盒，不是脚本编排。按可达性排序：

### 🥇 最易达：**机制独立 demo 集**
- 每个机制（粒子水、组装载具、破坏、武器）有自己的小展厅，进去就能玩
- 各 demo 之间无依赖，做完一个有一个，渐进交付
- 参考：Garry's Mod 早期、Box2D 自带 testbed
- 涌现深度：6/10（每个机制自洽，但跨机制涌现少）

### 🥈 中等：**物理沙盒（推荐路线）**
- 一个混合了破坏、流体、可拼装载具、武器的房间，玩家自由实验
- 无关卡设计，单纯的玩具盒；最大化展示统一物理涌现的乐趣
- 参考：Besiege Sandbox、Noita 创意模式、Teardown
- 涌现深度：9/10
- **个人推荐路线** —— 是涌现派最自然的归宿，且不需要关卡设计

### 🥉 进阶：**小型涌现挑战集**
- 一组小关，每关给一个目标（如"让那个箱子沉到水底"、"造一辆能爬上斜坡的车"、"用一颗炸弹解决三个敌人"）
- 玩家用统一物理工具集解决，多解开放
- 参考：Zachtronics 系列的物理化版本、World of Goo
- 涌现深度：10/10
- **风险**：需要关卡设计与谜题构思，工作量比沙盒大一档

> 不必现在决定走哪条。每个阶段 2 原型完成后再决定下一步走向。

## 8. 范围红线（明确不做）

- ❌ 完整剧情 / 角色塑造

- ❌ 存档 / 进度系统（除非某个 demo 必需）
